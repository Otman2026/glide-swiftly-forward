# خطة التحويل إلى منصة تجارية احترافية (Production-Ready)

## الوضع الحالي
النظام حالياً 92% مكتمل: 33 جدولاً مع RLS كامل، Multi-Tenant، 20 فهرساً للأداء، Soft-Disable للمستخدمين، HIBP مفعّل، Audit Log، Read-Only Mode. الوحدات الأساسية موجودة (عملاء، أوامر نقل، شحنات، رحلات، أسطول، سائقين، وقود، صيانة، مستودعات، مالية، HR، وثائق).

## الفجوات الفعلية المكتشفة
بعد فحص شامل، النواقص الحقيقية (وليست إعادة بناء ما هو موجود) هي:

1. **التصدير الموحّد**: أزرار Excel/CSV/PDF غير موجودة في معظم الجداول
2. **الطباعة**: لا يوجد Print Layout للفواتير وأوامر النقل
3. **GPS/الخرائط**: يوجد `trip_locations` لكن لا CRUD لأجهزة GPS ولا ربط بالمركبات ولا خريطة حية
4. **دورة الأعمال**: `transport_orders → trips → shipments → invoices` غير مربوطة تلقائياً (لا triggers/workflows)
5. **الأدوار المخصصة**: الأدوار ثابتة في `app_role` enum — لا يمكن للـ tenant إنشاء أدوار مخصصة
6. **KPI Dashboard**: لوحة إدارة عليا موجودة، لكن dashboard الشركة يفتقر لمؤشرات (ربحية الرحلة، استغلال المركبات، تكلفة الكم)
7. **التنبيهات الاستباقية**: `notifications` موجود لكن لا triggers تلقائية (انتهاء رخصة/تأمين/عقد)
8. **الأرشفة**: لا `archived_at` على الجداول التشغيلية
9. **إعادة تعيين كلمة المرور** من شاشة المستخدمين
10. **Backup**: لا يوجد endpoint تصدير بيانات الـ tenant

## المراحل (كل مرحلة قابلة للتسليم مستقلة)

### المرحلة 1 — البنية التحتية للأعمال (Backend)
- Migration: إضافة `archived_at` لكل الجداول التشغيلية + فهارس جزئية
- Migration: جدول `gps_devices` + ربط بـ `vehicles` + RLS
- Migration: جدول `custom_roles` + `custom_role_permissions` + دوال has_permission
- Migration: Triggers تلقائية للتنبيهات (انتهاء وثائق قبل 30/15/7 يوم)
- Migration: Trigger لتحويل `transport_order` معتمد → `trip` تلقائياً
- Migration: Trigger لإنشاء `invoice` من `trip` مكتمل

### المرحلة 2 — طبقة التصدير والطباعة (مشتركة)
- Hook موحّد `useExport` (Excel/CSV/PDF) عبر xlsx + jspdf
- مكوّن `<ExportBar>` قابل لإعادة الاستخدام
- Print layouts للفاتورة، أمر النقل، بوليصة الشحن
- تطبيقها على 15 شاشة رئيسية

### المرحلة 3 — GPS والخرائط الحقيقية
- شاشة CRUD لأجهزة GPS
- ربط جهاز ↔ مركبة
- خريطة حية (Leaflet + OpenStreetMap مجاناً)
- عرض المسار التاريخي لرحلة
- Server function لاستقبال إحداثيات من الأجهزة (`/api/public/gps-ingest` مع HMAC)

### المرحلة 4 — دورة الأعمال المتكاملة
- شاشة "دورة الطلب" تعرض المراحل مرئياً
- أزرار الانتقال بين المراحل مع validation
- حساب الربحية التلقائي لكل رحلة (إيراد - وقود - صيانة - راتب سائق نسبي)
- ربط `shipments` بـ `trips` بشكل many-to-many

### المرحلة 5 — KPI Dashboard احترافي
- 12 KPI card: إيراد الشهر، ربح، هامش، رحلات نشطة، استغلال أسطول %، متوسط تكلفة كم، أفضل 5 سائقين، أعلى 5 عملاء، وقود الشهر، صيانة معلّقة، فواتير متأخرة، شحنات في الطريق
- 4 charts (recharts موجود): إيراد/مصروف شهرياً، توزيع الرحلات، استهلاك الوقود، حالة الأسطول

### المرحلة 6 — الصلاحيات المخصصة
- شاشة إنشاء أدوار مخصصة لكل tenant
- Matrix صلاحيات (30 وحدة × 6 عمليات)
- دالة `has_permission(user, module, action)` تُستخدم في UI و RLS

### المرحلة 7 — إدارة المستخدمين المتقدمة
- زر "إعادة تعيين كلمة المرور" (server fn يستدعي admin API)
- زر "إعادة إرسال دعوة"
- سجل آخر تسجيل دخول
- تصدير بيانات tenant (JSON backup)

### المرحلة 8 — QA النهائي
- سكربت Playwright يمر على كل شاشة ويتحقق: الأزرار، النماذج، التصدير، الطباعة
- تقرير اكتمال نهائي
